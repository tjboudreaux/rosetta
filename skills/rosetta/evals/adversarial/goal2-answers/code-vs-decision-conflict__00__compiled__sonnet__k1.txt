(1) Current/production answer: Beacon (notification dispatcher) runs on MySQL in production. The production code (`notify_svc.py`, `DATASTORE = "mysql"`) and git commit 55dd66e ("notify_svc: provision MySQL (prod), retire Postgres POC", 2026-05-24) both confirm MySQL as the live datastore.

(2) Supporting resolution: ADR 0001 originally recorded Postgres as the chosen datastore (Accepted, 2026-02-01), but it was never amended after implementation. A licensing blocker on Postgres caused the team to pivot to MySQL at cutover; the decision record remained stale. ADR 0002 (Accepted, 2026-05-25) supersedes ADR 0001, anchors the current state to the production code and git history, and records MySQL as the shipped datastore. Per the Rosetta truth hierarchy, code and git history win over an un-amended decision record — so the answer is MySQL (ADR 0002), not Postgres (ADR 0001, stale/superseded).
