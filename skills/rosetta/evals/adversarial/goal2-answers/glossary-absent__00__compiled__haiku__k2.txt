The Cobalt service uses MySQL for its profile data storage, specifically on the mysql-prod-eu cluster. ADR 0002 establishes this as the accepted decision, supported by authoritative signals: the codebase (DSN "mysql://mysql-prod-eu/profiles" in billing_core.py), git commit 44ee55f (2026-03-15) that wired Cobalt to mysql-prod-eu, a dashboard export confirming MySQL as the backend engine, and operational evidence from incident review showing MySQL remained the datastore. A rejected distractor appears in ADR 0003: an unresolved TODO question about Cloud Spanner from 2026-01-09 that was never confirmed or adopted, making it a non-decision that conflicts with the established MySQL evidence.
