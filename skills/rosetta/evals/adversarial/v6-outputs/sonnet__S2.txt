The partner API currently uses API keys issued per integrator as its authentication method. This was decided in December 2025 ("partner API auth: API keys issued per integrator") and explicitly reaffirmed in June 2026 ("partner API auth: API keys remain supported for all integrators, reaffirmed"), both recorded in the history decision log. An earlier entry from February 2025 also noted "partner auth: adopt API keys," indicating this approach has been consistently applied. No subsequent decision superseded or replaced API keys for the partner API.
