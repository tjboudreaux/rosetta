The current authentication scheme for the external payments API is mutual TLS (mTLS) with client-certificate pinning. This replaced the prior OAuth2 bearer token scheme that had been in use throughout 2025. The change was recorded in the decision log entry dated 2026-06-11 (history/2026-06.md): "All external payment endpoints adopt mutual TLS with client-certificate pinning for PCI-DSS L1 compliance, replacing prior token-based access for these endpoints." The git-log.txt corroborates this with a commit on 2026-06-11 tagged `payments(us/enterprise): security`. The earlier OAuth2 bearer token scheme is documented repeatedly across the 2025 monthly history files (e.g., history/2025-01.md through history/2025-12.md), all stating "payments API auth: OAuth2 bearer tokens for all external endpoints," confirming the mTLS migration is a deliberate reversal of the previously consistent policy.
