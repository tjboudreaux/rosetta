The CURRENT authentication scheme for the external payments API is mutual TLS with client-certificate pinning, as documented in the engineering decision log from 2026-06-11. This represents a security upgrade for PCI-DSS L1 compliance, replacing the prior token-based access model that had been in use since early 2025. The mutual TLS with certificate pinning approach was adopted on June 11, 2026, making it the most recent and therefore current authentication scheme for external payment endpoints.
